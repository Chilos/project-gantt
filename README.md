# Project Gantt - Плагин диаграммы Ганта для Obsidian

Интерактивная диаграмма Ганта для управления проектами в Obsidian с возможностью визуального редактирования и синхронизации с текстом.

## Возможности

- 📊 **Интерактивная диаграмма Ганта** - Визуализация проектов, задач и этапов
- 🎨 **Умный выбор цвета шрифта** - Автоматический расчет контрастного цвета текста на основе яркости фона этапа
- 🎨 **Улучшенный дизайн этапов** - Количество дней слева, название и исполнитель вертикально справа для лучшей читаемости
- 🗓️ **Гибкие рабочие дни** - Настройка любых рабочих графиков: исключение дней недели, добавление/исключение конкретных дат, поддержка праздников
- 📅 **Дни недели в заголовках** - Отображение дней недели на русском (пн, вт, ср, чт, пт, сб, вс) для удобного планирования
- 🏃‍♂️ **Спринты и временные отрезки** - Организация работы по спринтам
- 🎯 **Этапы разработки** - Настраиваемые этапы (анализ, разработка, тестирование)
- 📍 **Мелстоуны** - Важные вехи проекта (ПСИ, РЕЛИЗ, Деплой)
- 👥 **Назначение ответственных** - На проекты, этапы и мелстоуны
- 🖱️ **Перетаскивание** - Визуальное изменение дат и длительности
- 📅 **Подсветка текущего дня** - Всегда видно где находитесь сейчас
- 🔄 **Синхронизация текста** - Автоматическое обновление при изменениях
- 📚 **Множественные диаграммы** - Независимое редактирование нескольких диаграмм в одной заметке
- 📋 **Режимы отображения** - Inline (компактно) или Multiline (каждый этап на отдельной строке)
- 🚧 **Ограничения перетаскивания** - Элементы не выходят за пределы временной шкалы

## Использование

### Базовый синтаксис

Создайте блок кода с типом `gantt` в любой заметке:

```
\`\`\`gantt
dates: 2024-01-01 | 2024-01-31

sprint: Спринт 1 | 2024-01-01 | 2024-01-14
sprint: Спринт 2 | 2024-01-15 | 2024-01-31

project: Проект 1
stage: Анализ | 2024-01-01 | 2024-01-05 | Иванов И.И. | #FF6B6B
stage: Разработка | 2024-01-06 | 2024-01-20 | Петров П.П. | #4ECDC4
stage: Тестирование | 2024-01-21 | 2024-01-28 | Сидоров С.С. | #45B7D1
milestone: ПСИ | 2024-01-28 | Руководитель | #FFD93D
milestone: РЕЛИЗ | 2024-01-31 | DevOps | #6C5CE7

project: Проект 2
stage: Подготовка | 2024-01-10 | 2024-01-15 | Команда А | #96CEB4
stage: Реализация | 2024-01-16 | 2024-01-30 | Команда Б | #FFEAA7
milestone: Деплой | 2024-01-30 | DevOps | #E17055
\`\`\`
```

### Гибкие настройки рабочих дней

Управляйте рабочими днями для точного планирования:
```
\`\`\`gantt
dates: 2025-07-21 | 2025-08-15
excludeWeekdays: 0,6          # Исключить выходные (воскресенье=0, суббота=6)
includeDates: 2025-07-26      # Суббота как рабочий день
excludeDates: 2025-08-12      # Вторник как праздник

project: Проект с гибким графиком
stage: Планирование | 2025-07-21 | 5
stage: Разработка | 2025-07-28 | 7
stage: Тестирование | 2025-08-06 | 3

sprint: Спринт 1 | 2025-07-21 | 2025-08-01
sprint: Спринт 2 | 2025-08-04 | 2025-08-15
\`\`\`
```
**Возможности:**
- **excludeWeekdays** - исключение дней недели (0=воскресенье, 6=суббота)
- **includeDates** - включение конкретных дат как рабочих
- **excludeDates** - исключение конкретных дат (праздники)
- **Приоритеты**: excludeDates > includeDates > excludeWeekdays > все дни рабочие

## Множественные диаграммы

В одной заметке можно создать несколько независимых диаграмм Ганта:
```
\`\`\`gantt
dates: 2024-01-01 | 2024-01-31
project: Проект A
stage: Этап 1 | 2024-01-01 | 2024-01-15 | Команда A | #FF6B6B
\`\`\`
```

```
\`\`\`gantt
dates: 2024-02-01 | 2024-02-29
project: Проект B  
stage: Этап 1 | 2024-02-01 | 2024-02-15 | Команда B | #27AE60
\`\`\`
```

**Преимущества:**
- Каждая диаграмма редактируется **независимо**
- Изменения в одной **НЕ влияют** на другие
- Можно планировать **параллельные проекты**

## Режимы отображения

### Inline режим (по умолчанию)
Все этапы проекта отображаются на одной строке:

```
\`\`\`gantt
project: Веб-приложение
stage: Анализ | 2024-01-01 | 2024-01-05 | Аналитик | #FF6B6B
stage: Разработка | 2024-01-06 | 2024-01-15 | Dev | #27AE60
\`\`\`
```

### Multiline режим
Каждый этап отображается на отдельной строке:

```
\`\`\`gantt
project: Мобильное приложение | multiline
stage: Исследование | 2024-01-01 | 2024-01-05 | UX | #E17055
stage: Разработка | 2024-01-06 | 2024-01-15 | Dev | #27AE60
stage: Тестирование | 2024-01-16 | 2024-01-20 | QA | #8E44AD
\`\`\`
```

**Когда использовать Multiline:**
- 🎯 Сложные проекты с множеством этапов
- 📝 Длинные названия этапов
- 👥 Важно видеть ответственных за каждый этап
- 📊 Детальное планирование с перекрывающимися задачами

## Ограничения перетаскивания

Элементы автоматически ограничиваются временными рамками диаграммы:

### Поведение:
- 🚫 **Этапы и мелстоуны** не выходят за пределы `dates`
- 🔴 **Визуальная индикация** - красная рамка при достижении границ  
- ⏱️ **Сохранение длительности** - этапы не сжимаются при упирании в границы
- 💾 **Автокоррекция** - если конец этапа упирается в границу, корректируется начало

### Примеры:

```
\`\`\`gantt
dates: 2024-01-01 | 2024-01-15

project: Тест границ
stage: Этап | 2024-01-05 | 2024-01-10 | Dev | #FF6B6B
# ↑ Этот этап нельзя перетащить до 1 января или после 15 января
\`\`\`
```

**Преимущества:**
- ✅ Предотвращение некорректных дат
- ✅ Сохранение целостности планирования  
- ✅ Интуитивное поведение интерфейса

### Синтаксис команд

#### Временные рамки
```
dates: YYYY-MM-DD | YYYY-MM-DD
```

#### Спринты
```
sprint: Название спринта | YYYY-MM-DD | YYYY-MM-DD
```

#### Проекты
```
project: Название проекта
project: Название проекта | multiline  # Каждый этап на отдельной строке
```

#### Этапы
```
stage: Название этапа | YYYY-MM-DD | YYYY-MM-DD | Ответственный | #Цвет
```

#### Мелстоуны
```
milestone: Название | YYYY-MM-DD | Ответственный | #Цвет
```

### Параметры

- **Название** - Обязательный текст
- **Даты** - Формат YYYY-MM-DD
- **Ответственный** - Необязательно, ФИО
- **Цвет** - Необязательно, HEX-код (#FF6B6B)

## Интерактивное редактирование

### Перетаскивание
- **Этапы** - Кликните и перетащите для изменения даты начала
- **Мелстоуны** - Перетащите для изменения даты

### Изменение размера этапов (Resize)
- **Только правый край** - Изменяет длительность этапа (увеличение/уменьшение дней)  
- **Курсор ↔** - Автоматически появляется при наведении на правый край
- **Плавное движение** - Этап плавно следует за движением мыши **БЕЗ СКАЧКОВ**
- **Стабильная дата начала** - Левый край и дата начала остаются неизменными
- **Real-time** - Изменения видны сразу во время перетаскивания
- **Финальная привязка** - Точное выравнивание по дням при отпускании мыши

### Автосохранение
Все изменения автоматически сохраняются в исходный текст заметки.

### Надежность перетаскивания
- ✅ **Абсолютная точность** - элементы остаются **точно** там где отпущены
- ✅ **Сохранение размеров** - этапы не сжимаются и не расширяются при перетаскивании
- ✅ **Отсутствие сдвигов** - математически точные преобразования позиции ↔ дата
- ✅ **Стабильность** - устойчивая работа при множественных операциях
- ✅ **Защита от перерисовки** - блокировка лишних обновлений интерфейса

## Настройки

В настройках плагина можно изменить:

- **Этапы по умолчанию** - Список стандартных этапов
- **Мелстоуны по умолчанию** - Список стандартных мелстоунов
- **Ширина ячейки** - Ширина одного дня (20-60px)
- **Высота строки** - Высота строки проекта (30-80px)

## Рабочие дни

По умолчанию учитываются только рабочие дни (Пн-Пт). Выходные дни пропускаются при расчете позиций.
